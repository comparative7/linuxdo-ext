# LinuxDo Auto-Browser Chrome Extension

## 背景 (Background)
为了保持在 LinuxDo 论坛的活跃度，需要一个自动化工具来模拟人类阅读帖子的行为。由于直接使用 API 或 Python 爬虫容易触发 Cloudflare 反爬机制并面临登录态管理的难题，我们决定开发一个基于 Manifest V3 (MV3) 的 Chrome 浏览器插件。该插件将寄宿在用户的真实浏览器环境中运行，完全共享登录态，并拥有真实的浏览器指纹。

## 目标 (Goals)
1. **自动浏览**：在 LinuxDo 帖子列表页（如首页或最新页），自动识别并点击未读帖子。
2. **拟人化阅读**：进入帖子详情页后，随机向下滚动页面，模拟真实的阅读停顿时间。
3. **循环调度**：帖子“阅读”完毕后，自动返回列表页，继续寻找下一个未读帖子，形成闭环。
4. **频率与防封控制 (Anti-Ban Limits)**：
   - **每日上限**：设置每天浏览帖子数量上限（如每日最多浏览 100 帖），跨天自动重置，达到上限后停止运行。
   - **劳逸结合**：每连续浏览 N 个帖子（如 10 个），强制自动暂停 M 分钟（如 15 分钟），休息结束后再继续下一轮。
   - **无未读轮询**：列表暂无未读时不直接停止，等待随机间隔后刷新列表再扫。
5. **状态控制**：提供 Popup「开始 / 停止」、运行中状态与今日计数；浏览参数在独立 Options 页调整，运行中/休息/等待时锁定编辑。

### 规划中的增强目标 (Planned Enhancements)
以下条目已纳入规格，**按批次实现**；实现前仍须先交技术更新计划并获批准。

6. **长帖早退 (#2)**：主题过长或单帖阅读过久时，按概率提前结束阅读并返回列表，避免极长帖拖死节奏。
7. **部分阅读 (#3)**：按概率在滚到页面深度约 40%–70% 时离开，不必每次滚到底。
8. **工具栏徽章 (#5)**：扩展图标显示今日帖数或简短状态（运行 / 休息 / 等待 / 达限），减少反复打开 Popup。
9. **跳过休息 / 立即扫描 (#9)**：休息或无未读等待期间，Popup 可一键跳过并立刻继续。
10. **今日足迹 (#10)**：记录并展示今日已读主题列表（标题、时间、链接），便于排查误点；可清空当日记录。
11. **HUD 增强 (#12)**：页面角标在列表扫描、休息、等待时也显示；休息/等待展示倒计时；按状态换色。
12. **正式图标 (#13)**：替换占位 PNG，提升工具栏与扩展管理页可识别度。

## 非目标 (Non-Goals)
1. **不执行写操作**：绝对不要实现自动发帖、自动回复或自动点赞功能（严格规避封号风险）。
2. **不处理复杂的登录**：假设用户在浏览器中已经手动登录了 LinuxDo。
3. **不引入前端框架**：Options / Popup 均为原生 HTML/CSS/JS，不引入 React/Vue 或打包工具。
4. **不做写操作周边**：不自动关注用户、不修改草稿、不操作通知已读以外的写接口。

## 核心架构方案 (Architecture)
*   **Manifest V3**：使用最新的 Chrome 扩展标准。
*   **Popup (popup.html / popup.js)**：负责向 Background 发送“启动/停止”及跳过暂停等指令；展示状态与足迹；提供「设置」入口打开 Options。
*   **Options (options/options.html / options.js)**：全部浏览参数（日限、休息、滚动、阅读策略等）的配置页；`options_ui.open_in_tab`；显式保存 / 恢复默认；运行中禁用编辑。
*   **Background Service Worker (background.js)**：作为中央调度器。负责记录当前状态（运行中/已停止）、日限与休息、徽章刷新，并在内容脚本发出信号时控制标签页跳转。
*   **Content Scripts (content.js)**：
    *   **列表页模式**：当注入到 `linux.do` 首页时，寻找帖子链接（需避开已读或置顶帖），点击进入。
    *   **详情页模式**：当注入到 `linux.do/t/*` 时，执行随机的平滑滚动；到底或触发早退/部分阅读后向 Background 发送完成信号。
    *   **页内 HUD**：只读浮动面板，`pointer-events: none`，不阻挡论坛交互。

## 增强需求明细 (Enhancement Specs)

> 推荐实现顺序：图标(#13) → 徽章(#5) → HUD(#12) → 跳过(#9) → 早退+部分阅读(#2+#3) → 足迹(#10)。  
> 建议每批单独 Commit；#2 与 #3 可同一批次。

### #13 正式图标
- **目标**：替换 `icons/icon16.png`、`icon48.png`、`icon128.png` 占位图。
- **设计**：主色贴近 Popup 主按钮（`#4a90d9`）；图形简洁（字母 LD 或抽象阅读意象）；16px 仍可辨认。
- **约束**：路径与 `manifest.json` 引用不变；静态 PNG；不做夜间双套、不做动态 icon。
- **验收**：
  - [ ] `chrome://extensions/` 加载无警告，工具栏与详情页图标更新。
  - [ ] 16px 尺寸下主色块/字母可辨认。

### #5 工具栏徽章
- **目标**：用 `chrome.action.setBadgeText` / `setBadgeBackgroundColor` 展示简要状态。
- **显示策略**：
  | 状态 | 文案 | 底色建议 |
  |------|------|----------|
  | 已停止 | 今日帖数（过大显示 `99+`） | 灰 |
  | 运行中 | 今日帖数 | 绿 |
  | 休息中 | `休`（首版可不绑秒级倒计时） | 橙 |
  | 等待新帖 | `等` | 蓝 |
  | 达日限 | `满` | 红 |
- **刷新时机**：启动/停止、读完一帖、进入/结束休息、进入/结束空闲轮询、扩展 `onStartup` / `onInstalled`。
- **权限**：不新增 manifest 权限。
- **验收**：
  - [ ] 开始后徽章为绿色数字；停止后变灰。
  - [ ] 读完一帖数字 +1；休息/等待文案切换正确。
  - [ ] Service Worker 回收重启后状态合理。

### #12 HUD 增强
- **目标**：角标覆盖更多运行态，而不只限于详情滚动。
- **形态**：
  | 场景 | 配色倾向 | 内容 |
  |------|----------|------|
  | 主题阅读中 | 蓝 | 今日统计 + 本主题已读/总数/本次新增 |
  | 列表扫描中 | 绿 | 运行中提示 + 今日统计 |
  | 休息中 | 橙 | `休息中 · mm:ss` + 今日统计 |
  | 等待新帖 | 蓝 | `等待新帖 · mm:ss` + 今日统计 |
  | 已停止 | — | 移除 HUD |
- **交互**：保持 `pointer-events: none`；倒计时由 content 本地 `setInterval` 刷新。
- **显示配置**（Options，默认全开）：
  | 键 | 默认 | 含义 |
  |----|------|------|
  | `hudEnabled` | `true` | 总开关；关闭后不渲染角标 |
  | `hudShowTitle` | `true` | 显示标题 `LinuxDo Bot` |
  | `hudShowStatus` | `true` | 显示运行状态（含休息/等待倒计时） |
  | `hudShowDaily` | `true` | 显示今日帖数与新读楼 |
  | `hudShowTopic` | `true` | 显示本主题进度（仅详情阅读） |
- **注意**：进入休息/等待时不应卸掉 HUD；停止或 Abort 时必须移除。配置变更经 `storage` 即时反映到已打开的 linux.do 页。
- **验收**：
  - [ ] 休息/等待时角标可见且倒计时跳动。
  - [ ] 停止后 HUD 消失；不遮挡页面可点区域。
  - [ ] Options 关闭总开关后角标消失；勾选取消单项后对应行不显示。

### #9 跳过休息 / 立即扫描
- **目标**：暂停态下用户可立刻恢复，不必干等 alarm。
- **UI**：Popup 增加按钮；仅在 `isResting || isWaitingForUnread` 时可用。文案：休息 →「跳过休息」；等待 →「立即扫描」。
- **消息**：Popup → BG `CMD_SKIP_PAUSE`；BG 复用现有 `endRestPeriod` / `endIdlePoll`（禁止重写另一套恢复逻辑）。
- **验收**：
  - [ ] 休息中跳过：立刻回列表并继续选帖。
  - [ ] 等待中立即扫描：列表刷新后重新扫描。
  - [ ] 非暂停态按钮禁用；未运行时无副作用。

### #2 长帖早退
- **目标**：主题过长或阅读超时 → 按概率提前发完成信号并回列表。
- **默认配置**（均可在 Popup 调整，需落在合理上下限内）：
  | 键 | 默认 | 含义 |
  |----|------|------|
  | `earlyExitEnabled` | `true` | 总开关 |
  | `earlyExitMaxPosts` | `80` | 总楼 ≥ 此值才可能按楼数早退 |
  | `earlyExitMaxMs` | `180000` | 单帖最长阅读时长（3 分钟） |
  | `earlyExitChance` | `70` | 满足条件后触发概率 % |
- **规则**：
  - 须至少完成少量滚动步后再允许楼数早退，避免进帖立刻退出。
  - 同一条件命中但未掷中概率后，该条件本帖不再重复掷骰；另一条件仍可触发。
  - 提前结束仍走 `EVT_POST_READ_COMPLETE`（携带 `newlyReadReplies` 与 `exitReason`）。
- **验收**：
  - [ ] 短帖不因楼数阈值误早退。
  - [ ] 将时长上限调极短后，长帖会提前结束并回列表。
  - [ ] 停止键仍能立刻中止。

### #3 部分阅读
- **目标**：一定概率在深度约 40%–70% 离开，不全程滚到底。
- **与 #2 优先级**（同帖只走一种提前离开路径）：
  ```
  超时早退 (#2) > 楼数早退 (#2) > 部分阅读目标达成 (#3) > 正常到底
  ```
- **默认配置**：
  | 键 | 默认 | 含义 |
  |----|------|------|
  | `partialReadEnabled` | `true` | 总开关 |
  | `partialReadChance` | `35` | 本帖采用部分阅读的概率 % |
  | `partialReadMinPct` | `40` | 目标深度下限 % |
  | `partialReadMaxPct` | `70` | 目标深度上限 % |
- **规则**：
  - 进帖时掷骰决定是否部分阅读；目标深度相对当前 `maxScrollY` 的百分比。
  - 极短帖（几乎无滚动空间）须至少完成最少滚动步/最短停留后再允许离开。
  - 与 #2 共用「提前完成」出口，并上报 `exitReason`。
- **验收**：
  - [ ] `partialReadChance=100` 且区间收窄时，几乎总在半程附近离开。
  - [ ] `chance=0` 时行为与改造前一致（仍可被 #2 打断）。
  - [ ] 今日新读楼统计仍正确。

### #10 今日足迹
- **目标**：持久化今日已读主题，Popup 可查看与清空。
- **存储**（`chrome.storage.local`，示意）：
  ```json
  {
    "browseHistory": {
      "date": "YYYY-MM-DD",
      "items": [
        {
          "topicId": "123",
          "title": "截断标题",
          "url": "https://linux.do/t/.../123",
          "at": 0,
          "newlyReadReplies": 0,
          "exitReason": "complete|early_timeout|early_posts|partial"
        }
      ]
    }
  }
  ```
- **规则**：跨天按日期键重置；最多保留最近 50 条；写入由 Background 在读帖完成时统一处理；Popup 可 `CMD_CLEAR_HISTORY` 清空当日（不影响 `dailyStats` 计数）。
- **UI**：折叠列表展示时间 + 标题；点击可打开对应主题 tab（只读跳转）。
- **验收**：
  - [ ] 每完成一帖足迹多一条；超 50 条丢最旧。
  - [ ] 跨天自动清空；手动清空不影响今日帖数统计。
