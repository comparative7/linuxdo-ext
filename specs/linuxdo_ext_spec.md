# LinuxDo Auto-Browser Chrome Extension

## 背景 (Background)
为了保持在 LinuxDo 论坛的活跃度，需要一个自动化工具来模拟人类阅读帖子的行为。由于直接使用 API 或 Python 爬虫容易触发 Cloudflare 反爬机制并面临登录态管理的难题，我们决定开发一个基于 Manifest V3 (MV3) 的 Chrome 浏览器插件。该插件将寄宿在用户的真实浏览器环境中运行，完全共享登录态，并拥有真实的浏览器指纹。

## 目标 (Goals)
1. **自动浏览**：在 LinuxDo 帖子列表页（如首页或最新页），自动识别并点击未读帖子。
2. **拟人化阅读**：进入帖子详情页后，随机向下滚动页面，模拟真实的阅读停顿时间。
3. **循环调度**：帖子“阅读”完毕后，自动返回列表页，继续寻找下一个未读帖子，形成闭环。
4. **状态控制**：提供一个简单的 Popup 弹窗，包含“开始”和“停止”按钮，能够随时介入控制。

## 非目标 (Non-Goals)
1. **不执行写操作**：绝对不要实现自动发帖、自动回复或自动点赞功能（严格规避封号风险）。
2. **不处理复杂的登录**：假设用户在浏览器中已经手动登录了 LinuxDo。
3. **不需要复杂的 UI**：除了 Popup 里的两个按钮，不需要复杂的配置页面。

## 核心架构方案 (Architecture)
*   **Manifest V3**：使用最新的 Chrome 扩展标准。
*   **Popup (popup.html / popup.js)**：负责向 Background 发送“启动/停止”指令。
*   **Background Service Worker (background.js)**：作为中央调度器。负责记录当前状态（运行中/已停止），并在内容脚本发出信号时，控制标签页的跳转（如从帖子页退回列表页）。
*   **Content Scripts (content.js)**：
    *   **列表页模式**：当注入到 `linux.do` 首页时，寻找帖子链接（需避开已读或置顶帖），点击进入。
    *   **详情页模式**：当注入到 `linux.do/t/*` 时，执行随机的平滑滚动，到底部后向 Background 发送 `POST_READ_COMPLETE` 信号。

## 验收标准 (Acceptance Criteria)
- [ ] 插件能成功在 Chrome `chrome://extensions/` 中通过开发者模式加载，无任何警告。
- [ ] 点击 Popup 的“开始”后，插件能在 `linux.do` 域名下自动开始工作。
- [ ] 在帖子内部的滚动表现出明显的随机性（例如滚动 500px 后停顿 3-7 秒）。
- [ ] 完成一个帖子后，能自动返回列表并进入下一个帖子。
- [ ] 点击 Popup 的“停止”后，所有自动化行为立刻终止。
